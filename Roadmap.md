# Roadmap Minecraft Holy Bot

На данный момент планируется 3 основных этапа развития программы.


## 1 Этап (текущий)

Этот этап развития кардинально меняет направление программы. 
Программа будет с открытым исходным кодом, кроссплатформенной, а также с другим названием.
В программе будут удалены простые DDOS боты(боты с методами BotJoiner, Memory и т.д.), умные боты будут переименованы в стресс-тест и добавлен менеджер ботов. В нем можно будет создавать одиночных(простых) ботов с плагинами для того чтобы автоматизировать игровой процесс на серверах, например, АФК-рыбалка.

Программа не получит обновление, пока все эти задачи не будут выполнены, так как я хочу чтобы пользователи получили высыкопроизводительное, удобное, легко расширяемое приложение для управления и создания ботов майнкрафт. Программа просто будет бесполезной, если, например, выпустить обновление, где будут простые боты, но не будут плагины.



## 2 Этап

К моменту начала работы над этим этапом в программе будет возможность создавать плагины, но этого мало, так как библиотека McProtoNet не дает полные возможности для работы с проткоолом Minecraft.


- Будут убраны библиотеки McProtNet.Protocol*. С помощью кодогенерации можно будет генерировать мультиверсионные клиенты майнкрафт.
- Программа и библиотеки SDK для неё получат обновления для того, чтобы пользоватся новыми возможностями McProtoNet.

## 3 Этап

Этот этап касается плагинов.

- Каждый плагин будет запускатся в отдельном процессе. Это нужно для того чтобы хост-программа не завершала работу аварийно, если в плагине произошла фатальная ошибка.
- Будет добавлена возможность управлять ботами через клиент Minecraft.
